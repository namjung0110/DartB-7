## 2강. 태블로연결


> **🧞‍♀️ 데이터 해석기는 어떤 경우 사용하나요?**

```
데이터 해석기(Data Interpreter)는 엑셀이나 CSV 파일처럼 구조가 깔끔하게 정리되지 않은 데이터 파일을 불러올 때 사용한다.
1.데이터 위에 제목이나 설명 문장이 있는 경우
2.표 사이에 빈 행이나 불필요한 행이 포함된 경우
3.보고서 형태로 작성되어 실제 데이터 영역이 명확하지 않은 경우
```

## 3강. 데이터연결과 데이터 유형


> **🧞‍♀️ 라이브와 추출의 차이가 무엇인가요? 어떤 경우 사용하나요?**


```
라이브 연결은 데이터 원본에 직접 연결하여 데이터를 실시간으로 가져오는 방식.
데이터가 변경되면 Tableau에서도 즉시 반영되기 때문에 항상 최신 데이터를 확인할 수 있다.
주로 실시간 데이터 분석이 필요한 경우나 데이터가 계속 업데이트되는 경우에 사용.
추출은 데이터 원본에서 필요한 데이터를 Tableau 내부 파일로 저장하여 사용하는 방식.
데이터를 따로 저장하기 때문에 분석 속도가 빠르고 오프라인 환경에서도 사용.
주로 대용량 데이터를 빠르게 분석해야 하거나 인터넷 연결이 어려운 환경에서 데이터를 사용해야 할 때 활용.
```



## 4강. 데이터결합과 관계

> **🧞‍♀️ 어떤 경우에 관계를, 어떤 경우에 조인을 사용하나요?**

```
관계는 여러 테이블을 논리적으로 연결하는 방식으로, 데이터를 실제로 합치지 않고 분석할 때 필요한 시점에 결합.
따라서 서로 다른 테이블을 유연하게 분석하거나 데이터 중복을 최소화하고 싶을 때 사용.

조인은 두 개 이상의 테이블을 하나의 테이블로 물리적으로 결합하는 방식.
분석 전에 데이터를 완전히 합쳐 하나의 데이터셋으로 사용해야 하거나 특정 조건에 맞는 데이터만 결합해야 할 때 사용.
```



## 5강. 데이터결합과 조인

<!-- 데이터 결합과 조인에 대해 알게 된 점을 적고, 아래 질문에 답해보세요 :) -->

주문 중 '반품된' 주문만을 가지고 분석을 진행하려고 합니다.

> **🧞‍♀️ 해당 목적 달성을 위해서 Sample store 데이터셋의 어떤 시트를 조인(혹은 릴레이션)해야 할까요? 조인키는 무엇인지, (inner, outer, left, right) 조인 유형은 무엇일지 논의해주세요.**
반품된 주문만 분석하려면 주문(Orders) 시트와 반품(Returns) 시트를 결합
조인키는 두 시트에 공통으로 들어 있는 주문 ID(Order ID)이다.
이 목적이 “반품된 주문만” 따로 보고 싶다는 것이므로, 가장 적절한 조인 유형은 Inner Join이다.
Inner Join을 하면 두 시트에 모두 존재하는 주문만 남기 때문에, 반품 테이블에 기록된 주문만 추출할 수 있다

## 6강. 데이터결합 혼합

데이터 결합 혼합은 서로 다른 데이터 소스를 함께 분석할 때 사용하는 방법이다.
예를 들어 하나는 Excel 데이터이고 다른 하나는 데이터베이스 데이터인 경우처럼 서로 다른 위치에 있는 데이터를 하나의 분석에서 함께 사용할 수 있다.
데이터 혼합은 기본 데이터 소스와 보조 데이터 소스를 기준으로 데이터를 연결한다.

## 8강. 라이브와 추출

> **🧞‍♀️ 라이브와 추출 방법의 차이가 무엇인가요? 어떤 경우에 추출을 사용하면 좋을까요?**

```
라이브 연결은 데이터 원본에 직접 연결하여 데이터를 실시간으로 가져오는 방식.
데이터 원본이 변경되면 Tableau에서도 바로 반영되기 때문에 항상 최신 데이터를 확인
반면 추출은 데이터 원본의 일부를 Tableau 파일로 저장하여 사용하는 방식이다.
데이터를 따로 저장하기 때문에 분석 속도가 빠르고 오프라인 환경에서도 사용할 수 있다.
추출은 데이터 용량이 크거나 분석 속도를 높이고 싶을 때, 또는 인터넷 연결이 어려운 환경에서 데이터를 사용해야 할 때 활용하면 좋다.
```



**참고자료 : 온프레미스 데이터란?**

## 문제 2.

> **🧚푸앙이는 태블로를 사용하여 회사의 매출 데이터를 시각화하려고 합니다. 태블로에서는 데이터를 연결하는 두 가지 방식인 라이브 연결과 추출 연결이 있습니다. 두 방식 중 어떤 것을 사용할지 고민 중입니다. 다음의 일상생활 사례를 바탕으로, 어떤 상황에서 라이브 연결이 적합하고, 어떤 상황에서 추출 연결이 더 적합한지 설명하세요.**

1. 사례 1: 실시간 주식 시세 확인
주식 거래를 하고 있는 수금의왕 푸앙이는 주식 시세가 매 순간 변동하는 실시간 데이터를 필요로 합니다. 이 데이터를 기반으로 빠르게 결정을 내려야 합니다.




```
라이브 연결이 적합하다.

주식 시세는 실시간으로 계속 변하는 데이터이기 때문에 항상 최신 정보를 확인할 수 있어야 한다. 
라이브 연결을 사용하면 데이터 원본과 직접 연결되어 데이터가 변경될 때마다 Tableau에도 즉시 반영되므로 실시간 데이터 분석에 적합하다.
```



2. 사례 2: 푸앙이는 저번 주 제주도로 여행을 갔었어요. 비행 중에 예산 계획을 세우려 하는데, 인터넷 연결이 불안정할 것 같아요. 여행 전 미리 예산 데이터를 다운로드해서 오프라인에서도 사용할 수 있으면 좋겠습니다.



```
추출 연결이 적합하다.

비행 중에는 인터넷 연결이 불안정하거나 사용할 수 없는 경우가 있기 때문에 데이터를 미리 다운로드하여 사용하는 것이 좋다. 
추출 연결은 데이터를 Tableau 파일로 저장하여 사용하기 때문에 오프라인 환경에서도 데이터를 확인하고 분석할 수 있다.
```
